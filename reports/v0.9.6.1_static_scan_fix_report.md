## 2. 低风险确定性修复

1. **tokens.css**：`@media (prefers-reduced-motion:reduce)` → `@media (prefers-reduced-motion: reduce)`。仅空格规范化，视觉语义不变。
2. **App.vue**：删除 `import { RouterView, RouterLink } from 'vue-router'`。这两个组件由 Vue Router 插件全局注册，template 中仍可解析；以 `npm run build` 验证。
3. **main.py**：删除 lifespan `yield` 之后的占位 `pass`，替换为注释 `# Shutdown: nothing to clean up yet`。startup 中 `except Exception: pass` 的语法占位 pass 未动。
4. **redaction.py**：将 `if reject: if 'content' in result:` 合并为 `if is_reject and 'content' in result:`。两个条件均为无副作用的字典读取，短路语义不变。

## 3. Security Hotspot 审查闭环

### 3.1 model_gateway/service.py 中的 172.29.128.1 — fixed

- **env override**：`WANWEI_OPENAI_COMPATIBLE_BASE` 环境变量始终优先，生产部署必须显式配置。
- **dev fallback**：`http://172.29.128.1:8084/v1` 仅作为 WSL → Windows host 本地 llama.cpp 的开发 fallback 保留，已加注释与 NOSONAR 标记。
- **production boundary**：真实（非 dry-run）smoke 调用前必经 `validate_external_url()` SSRF 校验；未在 `WANWEI_OPENAI_COMPATIBLE_HOST_ALLOWLIST` 精确列出的私网主机会被拒绝（新增测试 `test_real_smoke_still_ssrf_guarded` 证明默认 fallback IP 本身也被 SSRF 拦截）。未放开 172.16.0.0/12 整段。
- **用户不能传 api_base**：`ModelGatewayTestIn` 无 `api_base` 字段，pydantic 默认忽略额外字段；服务端只使用 provider catalog 中的 api_base。新增测试 `test_client_cannot_inject_api_base` 锁定该行为。
- 新增测试文件：`backend/app/tests/test_model_gateway_config.py`（env override 生效、dev fallback 保持、api_base 注入被忽略、SSRF 仍拦截）。

### 3.2 security/ssrf.py denylist 常量 — reviewed safe

这些是 RFC 保留/私网/回环/链路本地/CGNAT/组播/保留网段的 SSRF 防护 denylist 常量，**有意不可配置**：若改为运行时可配置（env/config），攻击者或误配置即可整体削弱默认防线，把"默认拒绝"变成"默认可关"。按部署差异放行的正确通道是 `validate_external_url(url, allowlist=...)` 的精确主机 allowlist，而不是修改 denylist。已在代码中加解释注释与最小 NOSONAR 标记，防护逻辑零改动。现有 `test_ssrf.py` 已覆盖回环/私网/云元数据/IPv6 回环被拒。

### 3.3 test_ssrf.py 测试 IP — reviewed safe

`127.0.0.1`、`169.254.169.254`、`10.0.0.1`、`192.168.1.1`、`172.16.0.1` 是 intentional SSRF test vectors; hardcoded by design。它们必须与 denylist 常量精确对应才能起回归防护作用，配置化反而使测试失去意义。已加模块级与行级注释/NOSONAR，断言强度未降低。

### 3.4 test_rate_limit.py 的 1.1.1.1 — reviewed safe

Intentional deterministic test IP：仅作为内存 token-bucket 的 key，无任何网络流量。已加注释，测试语义不变。

- **hippo_lite.recall**（17→≤15）：用 monkeypatch 固定 capsule 数据，锁定 seed 打分、1-hop 传播分值（0.45 spread）、无匹配 fallback（0.2）、0-hop 行为、evidence_paths 结构；随后提取 `_build_edge_map` / `_seed_scores` / `_spread_one_hop` 三个 helper。公共 API 与返回结构不变。
