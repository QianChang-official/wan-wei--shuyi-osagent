## 2. 核心功能实现

### 2.1 Workflow Run 持久化（P0）

**问题背景**：

v0.9.4 之前，workflow runs 存储在模块级字典 `_RUNS = {}`，导致：
- 进程重启后所有 run 数据丢失
- 多进程部署时状态不同步
- 无 TTL 清理，内存持续增长
- 无法水平扩展

**解决方案**：

新增 `backend/app/workflow/persistence.py` 模块，实现：

1. **数据库表结构**：

```sql
CREATE TABLE workflow_runs (
    run_id TEXT PRIMARY KEY,
    trace_id TEXT NOT NULL,
    scenario TEXT NOT NULL,
    user_goal TEXT NOT NULL,
    status TEXT NOT NULL,
    dry_run INTEGER NOT NULL DEFAULT 1,
    created_at TEXT NOT NULL,
    completed_at TEXT,
    run_data TEXT NOT NULL,  -- 完整 JSON
    version TEXT NOT NULL,
    total_stages INTEGER,
    completed_stages INTEGER,
    skipped_stages INTEGER,
    latency_ms INTEGER,
    risk_level TEXT
);

CREATE INDEX idx_workflow_runs_created_at ON workflow_runs(created_at);
CREATE INDEX idx_workflow_runs_status ON workflow_runs(status);
CREATE INDEX idx_workflow_runs_scenario ON workflow_runs(scenario);
```

2. **核心 API**：

- `save_run(run_id, run_data)` - 保存 run 到数据库
- `get_run(run_id)` - 读取 run（优先数据库，fallback 内存）
- `list_runs(limit, offset, scenario)` - 列表和过滤
- `cleanup_old_runs(ttl_days)` - TTL 清理（默认 7 天）
- `get_storage_stats()` - 存储统计

3. **迁移策略**：

- 写：同时写入数据库和内存（双写）
- 读：优先从数据库读取，fallback 到内存
- 内存字典保留为 fallback（v0.9.6 可移除）

**新增端点**：

- `GET /workflow/runs` - 列表（支持分页和场景过滤）
- `POST /workflow/cleanup` - 手动触发 TTL 清理
- `GET /workflow/stats` - 存储统计

**测试覆盖**：

- `test_workflow_persistence_basic` - 基本读写
- `test_workflow_list_and_filter` - 列表和过滤
- `test_workflow_cleanup` - TTL 清理
- `test_workflow_storage_stats` - 统计功能
- `test_workflow_service_integration` - 服务层集成

### 2.3 Timezone-aware Datetime 迁移（P0）

**问题背景**：

项目中广泛使用 `datetime.utcnow()`，但该方法已在 Python 3.12+ 标记为废弃：

```python
# 旧代码（已废弃）
datetime.datetime.utcnow().isoformat()
```

**解决方案**：

新增 `backend/app/utils/datetime_utils.py` 模块：

```python
def utc_now() -> datetime:
    """返回 timezone-aware 的当前 UTC 时间"""
    return datetime.now(timezone.utc)

def utc_now_iso() -> str:
    """返回 ISO 8601 格式的 UTC 时间字符串"""
    return utc_now().isoformat()

def utc_now_iso_compact() -> str:
    """返回紧凑格式（无微秒，Z 结尾）"""
    return utc_now().replace(microsecond=0).isoformat().replace('+00:00', 'Z')

def utc_timestamp_legacy() -> str:
    """向后兼容格式（无时区后缀）"""
    return utc_now().replace(tzinfo=None).isoformat()
```

**迁移范围**：

已迁移以下文件：
- ✅ `backend/app/workflow/service.py`
- ✅ `backend/app/audit/service.py`
- ✅ `backend/app/memory_runtime/capsule_store.py`
- ✅ `backend/app/memory_arena/runner.py`
- ✅ `backend/app/main.py`

**测试验证**：

- `test_utc_now_is_timezone_aware` - 验证时区信息
- `test_utc_now_iso_format` - 验证 ISO 格式

### 2.4 FastAPI Lifespan 迁移（P1）

**问题背景**：

FastAPI `@app.on_event('startup')` 已废弃，建议使用 `lifespan` 事件处理器。

**解决方案**：

迁移到 lifespan 模式：

```python
from contextlib import asynccontextmanager

@asynccontextmanager
async def lifespan(app: FastAPI):
    # Startup
    try:
        from .init_db import main as init_db
        init_db()
    except Exception:
        pass
    try:
        from .workflow.persistence import init_workflow_persistence
        init_workflow_persistence()
    except Exception:
        pass

    yield

    # Shutdown (if needed in future)
    pass

app = FastAPI(
    title='宛委·枢忆 MemoryOps Autopilot Platform',
    lifespan=lifespan,
)
```

**效果**：

- ✅ 移除了 10 个 on_event 相关的 DeprecationWarning
- ✅ 符合 FastAPI 最新最佳实践
- ✅ 为将来的 shutdown 逻辑预留接口


### 6.2 多进程部署

v0.9.5 现在支持多进程部署：

```bash
# Gunicorn 多进程
gunicorn backend.app.main:app \
  --workers 4 \
  --worker-class uvicorn.workers.UvicornWorker \
  --bind 0.0.0.0:8000
```

所有进程共享同一个 SQLite 数据库，workflow runs 状态一致。

### 6.3 TTL 清理

建议定期运行清理任务（cron）：

```bash
# 每天凌晨 2 点清理 7 天前的 runs
0 2 * * * curl -X POST http://localhost:8000/workflow/cleanup?ttl_days=7
```
